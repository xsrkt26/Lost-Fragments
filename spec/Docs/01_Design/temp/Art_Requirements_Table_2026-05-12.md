## 二、交付规范

| 项目 | 要求 |
| --- | --- |
| 文件格式 | 游戏内使用 PNG；需要透明背景的物品、饰品、道具、VFX 单帧均交付透明 PNG。 |
| 源文件 | 每个资产保留 PSD/KRA/AI 等可编辑源文件，图层命名需能看出结构，例如 `base`、`shadow`、`highlight`、`fx`。 |
| 命名建议 | UI：`assets/ui/v4/ui_<module>_<name>.png`；物品：`assets/items/icons/item_<id>.png`；饰品：`assets/ornaments/icons/orn_<id>.png`；道具：`assets/tools/icons/tool_<id>.png`；VFX：`assets/vfx/<effect>_<state>_<frame>.png`。 |
| 图标源尺寸 | 物品、饰品、道具建议源尺寸 512x512，导出 256x256 和 128x128 两档；需在 40x40 小图标尺寸下仍能读出轮廓。 |
| 局内物品显示 | 当前 `ItemUI` 以 100x94 为单格基础尺寸，多格物品按格子扩展。图标需能居中展示，也要允许后续升级为整张卡面。 |
| UI 切片 | 已有 v3 参考尺寸：整体布局 1440x1024、背包面板 809x936、背包格子 723x684、捕梦网 533x566、数值区 200x342、饰品栏 710x64、头像 311x342、垃圾桶 82x97。 |
| 状态表现 | 可交互 UI 至少提供普通、悬浮、按下、禁用；格子至少提供普通、可放置、不可放置、占用、锁定。 |
| 风格统一 | 关键词：梦境、碎片、旧物、轻微破损、污染黑紫、机械铜/铁、梦种绿色生命感。避免每个系统颜色过度割裂。 |
| 优先级 | P0：当前可玩闭环必须；P1：正式内容扩充必须；P2：临时候选或后续流派；P3：调试/开发辅助。 |
