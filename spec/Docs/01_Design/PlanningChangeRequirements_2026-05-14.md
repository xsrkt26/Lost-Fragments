## 二、全局术语变更

### 1. 梦值命名统一

正式资源名统一为 `梦值`，物品捕梦成本统一为 `梦值消耗`。

实现要求：

- UI、Tooltip、日志、测试名称和数据表解析不再显示 `San`、`san值` 或旧英文资源名。
- 若代码或旧存档仍保留旧字段名，需要在读取层做兼容，但对外展示必须转换为 `梦值`。
- 高消耗物品、梦值回复、梦值扣除、梦值归零结束局内游戏，都使用同一资源模型。

### 2. 撞击链术语替换

旧表述 `撞击链`、`链结束`、`链长` 统一替换为：

- `本次撞击结算`
- `本次撞击结算全部停止后`
- `本次命中物品数`
- `本次命中机械物品数`
- `本次转向传动次数`

实现要求：

- 代码可以继续保留历史类名，但对外文案、数据字段、测试描述应逐步迁移到新术语。
- 所有“每条链限 1 次”的语义改为“每次撞击结算限 1 次”。
- 所有“链长”收益改为按本次撞击结算的命中数计算。

### 3. 梦境之种命名

正式名称为 `梦境之种`，简称 `梦种`。

实现要求：

- UI 展示使用 `梦境之种` 或 `梦种`。
- 当前 Excel 中仍可能残留 `梦之种` 标签，该标签应作为旧别名归一化处理，不应作为新的展示文案。
- 与播种、升级、变大、4x4 终端相关的逻辑都归入梦境之种体系。

## 三、撞击结算需求

### 1. 撞击源与独立结算上下文

每个发起撞击的物品都是一个撞击源。每个撞击源创建一次独立的 `本次撞击结算`。

本次撞击结算至少记录：

- 已命中的物品集合。
- 本次命中物品数。
- 本次命中机械物品数。
- 本次转向传动次数。
- 待结算的普通传导分支。
- 待结算的机械传动分支。
- 本次是否已经触发某些限次效果。

实现要求：

- 同一个物品在同一次撞击结算中最多被命中一次。
- 如果某个分支即将命中已经命中过的物品，该分支立即停止，不触发 `被撞`，不增加命中数。
- 同一触发窗口中多个物品同时发起撞击时，按更靠近左上角优先；同一行内更靠左优先。
- 当前撞击源的本次撞击结算完全结束后，再结算下一个撞击源。

### 2. 普通撞击

普通撞击由物品向自身当前方向发起。

实现要求：

- 普通撞击只命中该方向上紧贴的物品。
- 若被命中物品方向与撞击方向相同，继续向同方向传导。
- 若目标为空、目标方向不同、目标已被本次结算命中过，则当前分支停止。

### 3. 机械传动

机械流新增正式关键词：

| 关键词 | 需求 |
| --- | --- |
| 机械撞击！ | 只会命中紧贴方向上的机械物品；目标为非机械或空格时分支停止。 |
| 向左传动！ | 从当前物品的左侧方向发起一次机械撞击；左侧相对于当前物品朝向计算。 |
| 向右传动！ | 从当前物品的右侧方向发起一次机械撞击；右侧相对于当前物品朝向计算。 |
| 双向传动！ | 从当前物品左侧和右侧各发起一次机械撞击。 |
| 全向传动！ | 从当前物品向所有相邻机械物品各发起一次机械撞击，只用于后期或严格限次内容。 |

实现要求：

- 机械传动产生的分支仍属于当前这一次撞击结算。
- 机械撞击命中机械物品后，仍然触发该物品的 `被撞` 效果。
- 机械撞击不能穿过非机械物品。
- 生成分支的物品效果默认每个物品每次撞击结算最多触发 1 次，除非效果明确写明其他规则。
- 污染只复制收益段，不复制撞击、机械撞击、向左传动、向右传动、双向传动、全向传动等生成分支的传动段。

## 四、污染体系变更

### 1. 移除污染反噬/梦值扣除口径

`60daa3d` 和 `bfe2f51` 移除了 `隔离箱` 以及“污染层数导致的梦值扣除效应”相关口径。

正式需求：

- 当前污染规则只保留“污染层数让物品结算效果额外触发 n 次”的核心逻辑。
- 不存在“污染层数自动导致梦值扣除”的全局反噬规则。
- 不再需要 `隔离箱` 这类专门减少污染扣梦值效应的物品。
- 若代码、测试或数据中仍存在 `隔离箱`、`isolation_box`、`pollution backlash`、污染导致自动扣梦值等实现，需要移除或标为旧数据兼容。

## 五、梦境之种需求

### 1. 基础规则

梦境之种是通过播种生成、通过等级成长扩大体型、通过撞击结算获得收益的长期构筑核心。

正式规则：

1. 在空格上播种生成 1 个 1 级、1x1 的梦境之种。
2. 对已经存在的梦境之种播种时，该梦境之种等级 +1。
3. 梦境之种每 10 级尝试变大一次。
4. 梦境之种最大为 4x4。
5. 升级变大时空间不足，梦境之种会从背包中掉出。
6. 掉出背包的梦境之种需要重新放入背包，才继续参与结算。
7. 4x4 后继续升级不会再次变大。

正式形态：

| 形态 | 等级范围 | 格子 | 被撞收益 |
| --- | --- | --- | --- |
| 1级形态 | 1-9 | 1x1 | +当前等级 x1 分 |
| 2级形态 | 10-19 | 2x2 | +当前等级 x2 分 |
| 3级形态 | 20-29 | 3x3 | +当前等级 x4 分 |
| 4级形态 | 30 及以上 | 4x4 | +当前等级 x8 分 |

实现要求：

- 旧版 5x5 终端不再是当前正式需求。
- 旧版 1-5 级固定形态不再匹配当前策划；代码需要支持真实等级阈值 10/20/30。
- 变大失败应产生可监听事件，供 `种子保险` 等饰品结算。
- 梦境之种本体不可直接捕梦获得。

### 2. 正式梦境之种物品池

除 4 个梦境之种基础形态外，当前正式梦境流物品为 15 个：

| 类型 | 物品 |
| --- | --- |
| 前期过渡 | 梦土铲、小喷壶、空心果壳、种子铃铛、月露灯、育苗托盘 |
| 核心物品 | 根系导管、温室灯、嫁接刀、蔓生罗盘、生态瓶、根网枢纽 |
| 后期补强 | 月冠温床、星根祭坛、巨木回声 |

关键实现点：

- 梦土铲：基础播种入口。
- 小喷壶：沿所指方向升级第一个梦境之种，未命中时给小额分。
- 空心果壳：从背包丢弃时在原位置或相邻空格播种。
- 月露灯：每捕梦 4 次升级随机梦境之种，无梦境之种时随机播种。
- 温室灯：每捕梦 5 次升级最高等级梦境之种，并在因此变大时给分。
- 蔓生罗盘、星根祭坛：依赖 `本次撞击结算全部停止后` 触发。
- 巨木回声：每个局内关卡第一次 4x4 梦境之种被撞后，从该梦境之种发起一次撞击。

## 六、机械流派需求

### 1. 正式机械物品池

当前正式机械物品为 15 个：

| 类型 | 物品 |
| --- | --- |
| 前期过渡 | 铁珠、小齿轮、传动皮带、左传动弯管、右传动弯管、制动片 |
| 核心物品 | 齿条、差速器、双轴转轮、蓄能飞轮、曲轴、计数轮 |
| 后期补强 | 中央引擎、终端计算机、星环轴承 |

实现要求：

- `顺转弯管`、`逆转弯管` 是旧名，正式名为 `左传动弯管`、`右传动弯管`。
- 曲轴正式效果使用 `向右传动！`。
- 机械收益围绕本次命中机械物品数、转向传动次数、双向传动和结算停止后的奖励展开。
- 星环轴承每个局内关卡第一次被撞触发全向传动，且这些分支不能再次触发星环轴承。
- `永动轮` 等会再次触发根源撞击、容易造成循环的候选物品不进入当前正式池。

### 2. 机械结算边界

实现必须覆盖以下边界：

- 同一次撞击结算同一个物品最多被命中 1 次。
- 机械分支不能穿过非机械物品。
- 向左/向右传动按当前物品朝向计算相对方向。
- `本次撞击结算全部停止后` 的效果只在该结算全部普通传导和机械分支结束后触发。
- 所有命中数统计都以“不同物品”为单位。

## 七、道具需求

道具系统当前仍按用户确认暂缓开发，但策划表已经从旧占位池整理为正式 15 个道具。后续恢复道具系统时，必须按本节作为正式范围。

### 1. 道具基础规则

- 道具是一次性物品，具有 `道具`、`罕见道具` 或 `稀有道具` 标签。
- 同名道具可以堆叠，无法拆分。
- 使用成功后堆叠数量 -1。
- 使用方式可以是点击使用，或拖拽到合法目标上使用。
- 目标非法时不应消耗道具。
- 道具不参与普通背包撞击结算。

### 2. 正式道具池

| 分类 | 道具 | 核心效果 |
| --- | --- | --- |
| 通用 | 小补丁 | 目标物品本局内价值 +3。 |
| 通用 | 梦值糖 | 下一次捕梦梦值消耗 -2，最低为 1。 |
| 通用 | 转向螺丝 | 目标物品顺时针旋转一次；若仍在背包内，+2 分。 |
| 通用 | 破弹珠 | 从目标物品发起一次撞击。 |
| 污染 | 黑墨滴 | 目标物品 +1 污染；若为废弃物额外 +1 污染。 |
| 污染 | 消毒喷雾 | 净化目标物品全部污染，每净化 1 层 +3 分。 |
| 污染 | 腐蚀酸 | 目标物品 +2 污染，并使其本局内价值 -5。 |
| 梦境之种 | 小水滴 | 目标是梦境之种则等级 +1；目标是空格则播种。 |
| 梦境之种 | 肥料包 | 空格播种；梦境之种等级 +2。 |
| 梦境之种 | 快速发芽剂 | 目标梦境之种等级 +3；若因此变大，+8 分。 |

## 八、饰品需求

### 1. 饰品总体规则

饰品不放入背包、不占背包格、不参与撞击空间传导，主要通过商店、事件、关卡奖励或 Boss 奖励获得。

实现要求：

- 同名饰品默认不能重复获得。
- 饰品获得后默认持续到整把游戏结束。
- 多个饰品同时触发时，按获得顺序结算。
- 如果多个饰品都修改同一数值，先结算“减少消耗”和“改变规则”，再结算“获得分数”和“获得梦值”。
- 稀有饰品从第 4 层开始进入奖励池；不设置独立“终局饰品”等级。

### 2. 正式饰品数量

当前 `饰品列表.xlsx` 总表为 56 个饰品：

- 原 50 个饰品已按梦境之种、机械传动和道具联动重新核对。
- 新增 6 个道具联动饰品：工具腰包、标本针盒、园艺工具包、回收挂钩、校准螺丝刀、万用工具箱。

道具联动饰品不构成独立大流派，而是分散到通用、污染、梦境之种与丢弃、机械与撞击、后期混合分类中。

### 3. 原 50 个饰品的关键修正

| 饰品 | 新需求 |
| --- | --- |
| 稳固背带 | 目标改为捕梦到占格数大于等于 4 的可抽取物品，不再写成大型梦种组件。 |
| 蜂蜜勺 | 触发对象收窄为苹果或美味烧鸡被丢弃。 |
| 温室玻璃 | 梦境之种等级提升时给分，达到 10/20/30 级时额外奖励。 |
| 种子保险 | 播种失败或梦境之种因变大空间不足掉出背包时给补偿。 |
| 收获篮 | 改为每个局内关卡第一次 4x4 梦境之种完成结算触发。 |
| 返青符 | 改为梦境之种达到 30 级或变为 4x4 时回复梦值，每关限 1 次。 |
| 链长计数器 | 正式显示名改为命中计数器。 |
| 齿轮油 | 机械物品成功传导或曲线传动命中机械物品时触发。 |
| 反冲片 | 一次撞击结算停止后，如果最后被撞物品是机械，从该物品反方向发起机械撞击，每关第一次。 |
| 万向轴承 | 每次撞击结算第一次命中机械物品时，该物品本次触发双向传动。 |
| 终端压力表 | 按本次命中机械物品数结算，不按全部命中物品数。 |
| 空梦奖杯 | 无目标分数的局内游戏不触发。 |

### 4. 新增道具联动饰品

| 饰品 | 分类 | 需求 |
| --- | --- | --- |
| 工具腰包 | 通用 | 每关第一次使用道具给分；每使用 3 个道具给额外分，每关限 1 次。 |
| 标本针盒 | 污染 | 使用污染或净化类道具给分；目标是废弃物时额外给分。 |
| 园艺工具包 | 梦境之种与丢弃 | 使用道具播种或升级梦境之种给分；因此变大时额外奖励。 |
| 回收挂钩 | 梦境之种与丢弃 | 使用丢弃相关道具给分；丢弃废弃物时提供有限梦值出口。 |
| 校准螺丝刀 | 机械与撞击 | 对机械物品使用道具后，奖励该物品下一次成功传导或曲线传动。 |
| 万用工具箱 | 后期混合 | 同一局内分别使用到物品、空格、捕梦区三类目标后给分并获得随机普通道具。 |

实现注意：

- 由于道具系统当前暂缓，51-56 号饰品可以先保留数据，但运行时效果应等道具系统恢复后接入。
- 如果奖励/商店已经能抽到 51-56 号饰品，而道具系统未实现，需要避免它们造成报错或不可解释的空效果；可临时标记为未启用，或以 Tooltip 明确显示但不触发。

## 十、实现验收标准

后续开发对齐这些策划提交时，至少满足以下验收标准：

1. 数据库加载后的正式物品池不包含 `隔离箱`。
2. UI 和 Tooltip 不再出现 `San`、`san值`、`撞击链`、`链长` 等旧口径。
3. `根源之梦` 每捕梦 5 次发起一次撞击，且进入统一撞击队列。
4. 普通撞击、机械撞击、向左/向右传动、双向传动、全向传动都基于同一个撞击结算上下文统计。
5. 同一次撞击结算中同一物品不会被重复命中。
6. 梦境之种最高体型为 4x4，等级阈值为 10/20/30。
7. 梦境之种变大空间不足时会掉出背包，并产生可供饰品监听的失败/掉出事件。
8. 机械正式物品名使用 `左传动弯管`、`右传动弯管`，不再使用 `顺转弯管`、`逆转弯管`。
9. 正式道具池为 15 个；道具系统恢复开发前，不实现旧候选道具。
10. 饰品总表以 56 个为准；非道具饰品先保持完整可运行，道具联动饰品随道具系统恢复开发。
11. 无目标分数的普通局内游戏不会触发依赖目标分数的饰品奖励。
12. 全量 GUT 和固定关键场景 headless 加载通过后，才能认为本轮需求实现完成。

## 十一、建议实现优先级

当前道具系统仍暂缓时，建议按以下顺序实现这些策划变更：

1. P0：清理旧术语、旧数据和旧物品池，尤其是 `San`、`撞击链`、`隔离箱`。
2. P0：重构撞击结算上下文，保证普通撞击和机械传动共享命中统计、限次和队列边界。
3. P1：将梦境之种从旧版 1-5 级/5x5 形态对齐到 1-30+ 等级与 1x1-4x4 体型。
4. P1：补齐机械传动正式关键词和 15 个机械物品效果。
5. P1：核对非道具饰品的 v1.2 修正，特别是命中计数器、终端压力表、种子保险、温室玻璃、返青符。
6. P2：在道具仍暂缓的前提下，保护 51-56 号饰品不报错、不污染奖励体验。
7. P2：待用户恢复道具需求后，按正式 15 个道具池实现道具栏、堆叠、目标校验、消耗和道具联动饰品。
