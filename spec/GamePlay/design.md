局内玩法

# 一、界面设计

界面包含的内容：
- **San 值条**：实时显示当前精神状态。
- **抽取动画播放窗口**：位于左侧，展示捕梦网与抽卡过程。
- **背包**：位于右侧，核心构筑区域。
- **垃圾桶**：允许玩家丢弃不需要的卡牌，丢弃动作可能触发特定效果。
- **饰品区**：右下角，存放对全局系统产生影响的装饰品。
- **角色头像**：左下角，代表玩家角色。

# 二、局内游戏基本流程

1、玩家开始一局局内游戏，进入主界面。

2、玩家可以点击左侧抽取界面，播放抽取动画，得到一件物品，物品会进入暂置区。

- **阶梯式 San 值扣除**：每次抽取物品都会扣除 San 值。初始扣除 **5** 点，后续每次抽取扣除 **5 + 1 * n** 点（n 为当前已抽取的次数）。
- **物品处理**：抽取到的物品会进入暂置区，玩家随后决定将其放入背包或丢弃。
- **丢弃逻辑**：如果玩家选择将卡牌拖入垃圾桶，会触发 `on_discard` 相关效果（部分卡牌或饰品可能在丢弃时产生收益）。
- **触发逻辑**：放置物品本身不再自动触发撞击。撞击将由特定事件触发（例如：抽到同名卡牌、特定饰品效果或环境事件）。

3、玩家可以将新物品置于背包的空格子中。如果满足特定触发条件（如抽到了同名物品），则会从背包中符合条件的坐标发起**撞击**。

4、重复上述步骤，直到玩家的 San 归 0，进入结算界面。玩家也可随时点击结束按钮中止并回到外部地图。

# 三、背包与物品

**物品基本设计**
物品属性如下：
- 名称
- **类别 (Category/Tags)**：如“运动”、“神秘”、“工具”等。不同类别的物品会受到特定饰品或全局规则的影响。
- 占格子情况 (Shape)
- 物品方向 (Direction)
- 附带效果 (Effects)
- **运行时 ID (runtime_id)**：用于逻辑与表现层的 1:1 稳定绑定。

**撞击！与连锁反应**
- **触发接口**：系统提供特定接口 `trigger_impact_at(pos)`，允许从背包中的任意坐标发起一次撞击。
- **同名连锁示例**：假定“棒球”卡牌的效果是“抽到此卡时，背包内所有棒球发起撞击”。此时系统会遍历背包，找到所有棒球实例，并分别从它们所在的位置按其朝向发起撞击链。

# 四、饰品与外部影响

**饰品区 (Ornaments)**
- 位于界面右下角，存放被动生效的装饰品。
- **系统性影响**：饰品可能改变全局规则。例如：
	- 降低 San 值消耗。
	- 增加特定类别（如“运动类”）物品的结算分数。
	- 改变撞击的传播距离或穿透属性。
